# Simple Whisper 详细教学文档

## 文档简介

本教学文档旨在为 Simple Whisper 语音转文字应用提供从零开始的详细使用指导。无论您是编程新手还是有经验的开发者，都能通过本教程快速掌握该工具的各项功能。

### 工具概览

Simple Whisper 是一个基于 OpenAI Whisper 模型的语音转文字工具，具有以下特点：

- **简单易用**：提供多种使用方式，满足不同用户需求
- **功能全面**：支持录音、转录、批量处理等核心功能
- **跨平台**：支持 macOS、Linux、Windows（部分功能）
- **性能优化**：支持 CPU、GPU 加速，多模型选择

## 第一章：环境准备与安装

### 1.1 系统要求

- **操作系统**：macOS 10.15+、Ubuntu 18.04+、Windows 10+（需额外配置）
- **Python 版本**：Python 3.8 或更高版本
- **内存**：至少 4GB RAM（推荐 8GB+）
- **存储空间**：至少 2GB 可用空间（用于模型存储）

### 1.2 检查 Python 环境

打开终端（Terminal），输入以下命令检查 Python 版本：

```bash
# 检查 Python 版本
python --version
# 或
python3 --version
```

如果显示 Python 3.8 或更高版本，可以继续下一步。如果没有安装 Python，请先安装 Python 3.8+。

### 1.3 克隆或下载项目

如果您还没有获取项目文件，可以通过以下方式获取：

**方法一：如果已在当前目录**
```bash
# 检查是否在正确目录
pwd
ls -la
```

**方法二：从其他地方复制**
```bash
# 假设项目在 ~/claude/simple-whisper-app
cd ~/claude/simple-whisper-app
```

### 1.4 一键安装（推荐）

项目提供了自动安装脚本，可以一键完成所有设置：

```bash
# 进入项目目录
cd /path/to/simple-whisper-app

# 运行安装脚本
./setup.sh
```

安装脚本会自动完成以下操作：
1. 创建 Python 虚拟环境
2. 激活虚拟环境
3. 安装所有依赖包
4. 显示完成提示

### 1.5 手动安装步骤

如果一键安装失败，可以按以下步骤手动安装：

```bash
# 步骤1：创建虚拟环境
python -m venv venv

# 步骤2：激活虚拟环境
# macOS/Linux:
source venv/bin/activate
# Windows:
venv\Scripts\activate

# 步骤3：升级 pip
pip install --upgrade pip

# 步骤4：安装依赖
pip install -r requirements.txt
```

### 1.6 安装系统依赖（macOS）

如果您在 macOS 上使用，可能需要安装 PortAudio：

```bash
# 使用 Homebrew 安装 PortAudio
brew install portaudio
```

如果遇到 "brew: command not found" 错误，请先安装 Homebrew：
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### 1.7 验证安装

安装完成后，运行系统测试验证安装是否成功：

```bash
# 运行系统测试
./test_system.sh
```

测试脚本会：
1. 检查 Python 环境
2. 检查依赖包
3. 测试音频设备
4. 测试 Whisper 模型
5. 进行一次 2 秒的录音测试

如果看到 "SYSTEM TEST COMPLETE" 和绿色对勾 ✓，说明安装成功。

## 第二章：快速上手 - 第一个录音

### 2.1 激活虚拟环境

每次使用前需要激活虚拟环境：

```bash
# 进入项目目录
cd /path/to/simple-whisper-app

# 激活虚拟环境
source venv/bin/activate

# 验证激活（命令行前缀会出现 (venv)）
# (venv) user@computer simple-whisper-app $
```

### 2.2 查看音频设备

在进行录音前，先查看可用的麦克风设备：

```bash
python simple_whisper.py --list-audio-devices
```

输出示例：
```
Available audio input devices:
------------------------------------------------------------
  [0] "External Microphone"
      Input channels: 1, Sample rate: 48000.0
  [5] "MacBook Pro麦克风" (default)
      Input channels: 1, Sample rate: 48000.0
  [7] "Aggregate Device"
      Input channels: 3, Sample rate: 48000.0
------------------------------------------------------------
Default input device ID: 5
```

记下您想要使用的设备 ID（比如 MacBook Pro麦克风是 ID 5）。

### 2.3 进行第一次录音

让我们进行一个简单的 5 秒测试录音：

```bash
python simple_whisper.py --record --duration 5 --model tiny
```

操作步骤：
1. 命令执行后，会显示 "Recording for 5.0 seconds..."
2. 立即对着麦克风说话（可以说："这是一个测试录音"）
3. 5 秒后自动停止并开始转录
4. 完成后会显示转录结果

### 2.4 查看输出文件

录音完成后，会自动生成两个文件：

```bash
# 查看生成的文件
ls -la recording_*.wav recording_*.txt
```

文件命名规则：
- 音频文件：`recording_YYYYMMDD_HHMMSS.wav`
- 转录文件：`recording_YYYYMMDD_HHMMSS_transcription.txt`

查看转录内容：
```bash
cat recording_*.txt
```

### 2.5 理解输出信息

转录完成后，会显示以下信息：

```
==================================================
TRANSCRIPTION RESULT:
==================================================
这是一个测试录音
==================================================
Transcription saved to: recording_20260221_143022_transcription.txt

Summary:
- Audio: recording_20260221_143022.wav
- Transcription: recording_20260221_143022_transcription.txt
- Model: tiny
- Language: zh
```

重要信息：
- **模型**：使用的 Whisper 模型
- **语言**：检测到的语言（zh 表示中文）
- **文件路径**：保存的文件位置

## 第三章：核心功能详细教学

### 3.1 录音功能详解

#### 3.1.1 定时录音

```bash
# 录制 30 秒音频
python simple_whisper.py --record --duration 30

# 录制 2 分钟（120 秒）
python simple_whisper.py --record --duration 120

# 录制 0.5 小时（1800 秒）
python simple_whisper.py --record --duration 1800
```

#### 3.1.2 手动停止录音

如果不指定时长，可以手动停止录音：

```bash
python simple_whisper.py --record
```

操作：
1. 看到 "Recording until interrupted (Ctrl+C)..." 提示
2. 开始说话
3. 按 `Ctrl+C` 停止录音
4. 自动开始转录

#### 3.1.3 指定输出文件名

```bash
# 指定音频文件名
python simple_whisper.py --record --duration 10 --output-audio my_audio.wav

# 指定转录文件名
python simple_whisper.py --record --duration 10 --output-text my_transcription.txt

# 同时指定两个文件名
python simple_whisper.py --record --duration 10 --output-audio audio.wav --output-text text.txt
```

### 3.2 麦克风选择功能

#### 3.2.1 查看所有麦克风

```bash
python simple_whisper.py --list-audio-devices
```

#### 3.2.2 使用特定麦克风

```bash
# 使用设备 ID 0 的麦克风
python simple_whisper.py --record --duration 10 --input-device 0

# 使用设备 ID 5 的麦克风（MacBook Pro内置）
python simple_whisper.py --record --duration 10 --input-device 5

# 使用默认麦克风（自动选择）
python simple_whisper.py --record --duration 10
```

#### 3.2.3 验证麦克风工作

```bash
# 测试特定麦克风（2秒测试）
python simple_whisper.py --record --duration 2 --input-device 5 --model tiny
```

### 3.3 模型选择详解

#### 3.3.1 可用模型对比

| 模型 | 参数大小 | 速度 | 准确度 | 推荐场景 |
|------|---------|------|--------|----------|
| tiny | 39M | ⚡️ 很快 | 较低 | 快速测试、短语音 |
| base | 74M | 快 | 中等 | 日常使用、对话 |
| small | 244M | 中等 | 良好 | 会议、访谈 |
| medium | 769M | 慢 | 优秀 | 重要会议、高准确度 |
| large | 1550M | 🐌 很慢 | 最佳 | 研究、最高要求 |

#### 3.3.2 选择合适模型

```bash
# 快速测试
python simple_whisper.py --record --duration 5 --model tiny

# 日常使用
python simple_whisper.py --record --duration 30 --model base

# 会议记录
python simple_whisper.py --record --duration 600 --model small

# 重要访谈
python simple_whisper.py --record --duration 1800 --model medium

# 研究用途
python simple_whisper.py --record --duration 10 --model large
```

## 第四章：高级使用方式

### 4.1 交互式界面

对于不熟悉命令行的用户，可以使用交互式界面：

```bash
python interactive_whisper.py
```

交互式界面会引导您完成以下步骤：
1. **选择模式**：录音模式或文件转录模式
2. **选择麦克风**：从列表中选择音频设备
3. **选择模型**：选择 Whisper 模型大小
4. **选择计算设备**：CPU、GPU 或自动选择
5. **选择语言**：指定语言或自动检测
6. **设置时长**：如果是录音模式
7. **开始处理**：确认后开始录音/转录

### 4.2 快捷脚本

#### 4.2.1 快速录音脚本

```bash
# 查看帮助
./quick_record.sh --help

# 录制 10 秒（使用默认模型）
./quick_record.sh -d 10

# 录制 60 秒，使用 small 模型
./quick_record.sh -d 60 -m small

# 录制 5 分钟，指定输出文件名
./quick_record.sh -d 300 -o my_meeting
# 会生成：my_meeting.wav 和 my_meeting_transcription.txt
```

#### 4.2.2 文件转录脚本

```bash
# 查看帮助
./transcribe_file.sh --help

# 转录文件（自动检测语言）
./transcribe_file.sh audio.wav

# 转录文件，指定模型
./transcribe_file.sh -m small meeting.mp3

# 转录文件，指定语言
./transcribe_file.sh -l zh interview.wav

# 转录文件，指定输出文件名
./transcribe_file.sh -o transcript.txt audio.m4a
```

#### 4.2.3 会议录制脚本

```bash
# 启动会议录制向导
./record_meeting.sh
```

脚本会询问：
1. 会议名称（用于生成文件名）
2. 会议时长（小时/分钟）
3. 会议语言
4. 确认后开始高质量录制

### 4.3 批量处理

#### 4.3.1 基本批量转录

```bash
# 转录文件夹中的所有音频文件
python batch_transcribe.py ./audio_files

# 指定输出文件夹
python batch_transcribe.py ./audio_files --output-folder ./transcripts

# 指定模型
python batch_transcribe.py ./audio_files --model small
```

#### 4.3.2 高级批量选项

```bash
# 递归搜索子文件夹
python batch_transcribe.py ./audio_files --recursive

# 指定文件格式
python batch_transcribe.py ./audio_files --extensions "wav,mp3,m4a"

# 指定语言
python batch_transcribe.py ./audio_files --language en

# 使用 GPU 加速
python batch_transcribe.py ./audio_files --device cuda
```

#### 4.3.3 批量处理示例

假设有以下文件夹结构：
```
audio_recordings/
├── meeting_20240201.wav
├── interview_20240202.mp3
└── notes_20240203.m4a
```

批量转录命令：
```bash
# 创建转录文件夹
mkdir -p transcripts

# 批量转录
python batch_transcribe.py audio_recordings --output-folder transcripts --model base --language zh

# 查看结果
ls transcripts/
# meeting_20240201_transcription.txt
# interview_20240202_transcription.txt
# notes_20240203_transcription.txt
```

## 第六章：故障排除与优化

### 6.1 常见问题解决

#### 问题1：无法找到音频设备

**症状**：
```
Error: No audio input devices found
```

**解决方法**：
```bash
# 检查系统音频设置
python -c "import sounddevice as sd; print(sd.query_devices())"

# macOS 安装 PortAudio
brew install portaudio

# Ubuntu/Debian
sudo apt-get install portaudio19-dev python3-pyaudio
```

#### 问题2：模型加载失败

**症状**：
```
Error loading model: [错误信息]
```

**解决方法**：
```bash
# 检查网络连接（第一次需要下载模型）
# 模型下载位置：~/.cache/whisper/

# 手动下载模型（如果自动下载失败）
# 可以从 Hugging Face 手动下载并放入缓存目录

# 尝试更小的模型
python simple_whisper.py --record --duration 2 --model tiny
```

#### 问题3：转录结果为空

**症状**：转录文件只有 "--- Detailed Segments ---"

**解决方法**：
```bash
# 检查音频是否有声音
python -c "
import soundfile as sf
data, samplerate = sf.read('recording.wav')
print(f'音频长度: {len(data)/samplerate:.1f}秒')
print(f'最大音量: {abs(data).max():.3f}')
"

# 增加录音音量
# 说话时离麦克风更近
# 使用更高质量的模型
python simple_whisper.py --audio recording.wav --model small
```

#### 问题4：转录速度太慢

**症状**：处理时间过长

**解决方法**：
```bash
# 使用更小的模型
python simple_whisper.py --audio file.wav --model tiny

# 使用 GPU 加速
python simple_whisper.py --audio file.wav --model base --device cuda

# 截短过长的音频
# 使用音频编辑软件分割长文件
```

### 6.2 性能优化建议

#### 硬件优化
- **CPU**：确保有足够的内存（8GB+）
- **GPU**：如果有 NVIDIA GPU，使用 `--device cuda`
- **Apple Silicon**：使用 `--device mps` 加速

#### 软件优化
- **模型选择**：根据需求选择最小可用模型
- **音频质量**：使用 16kHz 单声道 WAV 格式
- **批量处理**：一次处理多个文件，减少模型加载次数

#### 使用技巧
```bash
# 组合优化参数
python simple_whisper.py --record --duration 60 --model base --device mps --language zh

# 预处理长音频
# 使用 ffmpeg 分割长文件
ffmpeg -i long_audio.wav -f segment -segment_time 600 -c copy output_%03d.wav
```

### 6.3 质量优化建议

#### 提高转录准确度

#!/bin/bash
WATCH_DIR="/path/to/new_recordings"
PROCESSED_DIR="/path/to/processed"

inotifywait -m -e create "$WATCH_DIR" |
while read path action file; do
    if [[ "$file" =~ \.(wav|mp3|m4a)$ ]]; then

        python simple_whisper.py --audio "$WATCH_DIR/$file" --model base --output-text "$PROCESSED_DIR/${file%.*}.txt"

    fi
done
EOF

### 8.2 模型下载信息

Whisper 模型第一次使用时自动下载，位置在：
- **Linux/macOS**: `~/.cache/whisper/`
- **Windows**: `C:\Users\<username>\.cache\whisper\`

模型文件大小：
- tiny: ~150 MB
- base: ~290 MB
- small: ~970 MB
- medium: ~3.1 GB
- large: ~6.2 GB

### 8.3 音频格式支持

支持格式：
- **WAV**（推荐）：无损，兼容性最好
- **MP3**：有损压缩，文件较小
- **M4A**：Apple 音频格式
- **FLAC**：无损压缩
- **OGG**：开源格式

推荐使用 WAV 格式获得最佳效果。
